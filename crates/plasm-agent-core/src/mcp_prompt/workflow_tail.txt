     Also: **`plasm`** is **plan-only** (topology + expected shapes); it never performs live API calls. For reports, joins, writes, or multi-step work, prefer **one composed multi-line `plasm_program`** in **one** **`plasm`** call over many one-line calls. **Final roots** in one **`plasm`** call should be a **single** expression or one multi-line program; bare comma-separated **multi-root** programs belong on **`plasm_run`** after review. Never prefix final roots with `return`. Response order follows the final roots; execution order follows Plasm/runtime dependencies.
     **`plasm_run`**: same arguments as **`plasm`**. Call **only after** the **`plasm`** dry-run plan matches intent—this tool performs **live** HTTP/API execution (and may return run snapshot URIs).
     **Paging:** follow **`page(s0_pgN)`** / `_meta.plasm.paging` in the same logical session for more rows.
     **Run snapshots:** `plasm://…` URIs are MCP **`resources/read`** targets, not Plasm expressions — call **`resources/read`** for full JSON when the summary points there.
