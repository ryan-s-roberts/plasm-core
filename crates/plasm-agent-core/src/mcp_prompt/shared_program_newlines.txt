**`program` string (JSON):** For anything beyond a single-line expression, the string must contain **real newline characters (U+000A)** between physical lines—typically one `ident = …` binding per line, then final bare roots on their own line(s). **Spaces do not separate statements.** JSON/MCP may show these as `\n` on the wire; the server still expects actual newline characters after parsing. Self-check: splitting on `\n` yields one statement per line until comma-separated final roots (roots stay on their own line(s)). Good: `repo = e2(...)\ncommits = e1{p4=repo}.limit(20)\ncommits`. Bad: `repo = e2(...) commits = e1{p4=repo}.limit(20) commits`. **Tagged `<<TAG` heredocs** need a newline immediately after `<<TAG` on the opener line, then body lines, then a closing line whose trimmed text is `TAG` (or `TAG)` / `TAG,` / `TAG}`). Commas inside a heredoc body never split parallel roots.
