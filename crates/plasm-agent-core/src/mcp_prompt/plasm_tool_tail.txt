
For one direct read/search/get/relation/method/action whose result is already the answer, send one taught **`plasm_expr`**. For reports, joins, writes, markdown/html payloads, or any multi-step task, send **one multi-line `plasm_program`** in **one** **`plasm`** call.
**Postfix transforms** (`.limit`, `.sort`, `.aggregate`, `.group_by`, `[…]` projection, `.singleton()`, `.page_size`) apply to the **same** surface expression language: e.g. `e1{p4=repo}.limit(20)` is valid on one line without an extra binding. **Aggregates / group_by** use explicit output names in each spec: e.g. `commits.group_by(author, n=count)` and `orders.aggregate(total=sum(amount), lo=min(price))`.
**Brace predicates (`{…}`):** slots typed as **entity references** toward another resource expect a DOMAIN **constructor**, scalar identity fields, or **`anchor.<relation>`** — not an opaque full row unless its top-level identity fields match that entity’s keys (see schema `key_vars` / id).
**Continuation:** when a binding is a taught surface row, `next = repo.<relation>` continues that row’s Plasm (same as inlining the anchor expression before `.<relation>`); the plan keeps an edge from `repo` to `next`. Do not extend **projection/limit** chains (`…[fields]`, `.limit`, …) with `ident.<relation>` — bind a fresh surface node first.
Reuse **`logical_session_ref`** for follow-ups. Call **`plasm_context`** to **append** new `{api, entity}` picks when you learn more integrations, or with the **same** **`seeds`** when you only need a fresh symbol table.
