Plasm is **several MCP tools**; use what you need, in order. **`plasm_context`** opens one logical session; **`plasm`** returns a **dry-run plan** for programs inside it; **`plasm_run`** performs **live execution** after you review that plan; **`discover_capabilities`** finds **`api`** and **`entity`** values that match your intent when you do not already know them. **Skip `discover_capabilities`** when you already know every **`api`/`entity`** pair you need.
     **One user goal → one stable **`intent`** → one **`plasm_context`**.** `logical_session_ref` slots (`s0`, `s1`, …) are **per logical goal**, not per API. Federated example: `[{"api":"pokeapi","entity":"Pokemon"},{"api":"linear","entity":"Issue"}]` in **one** **`seeds`** array — not a second context open for the same task.
     **`discover_capabilities`**: **one call per user goal** with one **`intent`** string for the whole task (multi-API goals in that sentence). No keyword arrays, **`query`**/**`utterance`**, or per-integration splits. Reply is a fenced table (**`api`**, **`entity`**, **`description`**); merge rows into one **`plasm_context`** **`seeds`**. Retry only if entities are missing. Use **`typed: true`** when ambiguous.
     **`plasm_context`**: see the **`plasm_context`** tool description for seeds shape, teaching TSV output, and `_meta.plasm` continuity. **Steady state:** reuse **`logical_session_ref`** with **`plasm`** / **`plasm_run`** — do **not** repeat **`plasm_context`** with identical **`seeds`** every turn.
     **`plasm`**: pass **`logical_session_ref`** and one **`program`** string.
