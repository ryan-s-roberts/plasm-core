Plasm is **several MCP tools**; use what you need, in order. **`plasm_context`** opens one logical session; **`plasm`** returns a **dry-run plan** for programs inside it; **`plasm_run`** performs **live execution** after you review that plan; **`discover_capabilities`** finds **`api`** and **`entity`** values that match your intent when you do not already know them. **Skip `discover_capabilities`** when you already know every **`api`/`entity`** pair you need.
     **`discover_capabilities`**: pass **`query`** as one short string (what the user wants) or as several strings. The reply is a small fenced **table**: **`api`** (which integration), **`entity`** (which resource type there), **`description`**. Each row is one **capability pick** ‚Äî use them to fill **`plasm_context`**‚Äôs **`seeds`** array (same columns per object).
     **`plasm_context`**: pass a stable **`intent`** string and a non-empty **`seeds`** array. Each object is **one capability pick**: **`api`** names the integration, **`entity`** names the resource type (same shape as discovery rows). **List every pick your program needs on the first call**‚Äîif the task spans more than one integration, put **all** picks in the **same** **`seeds`** array; that is still **one** session and **one** program surface for **`plasm`** / **`plasm_run`**. You may use **`entry_id`** instead of **`api`** on each object. Do not shrink **`seeds`** to ‚Äúnarrow‚Äù the session, and do not rotate **`intent`** every user message.
     **Adding picks:** call again when you discover **additional** capability picks you did not know at open time (append new `{api, entity}` rows‚Äîstill additive).
     **Symbols:** authoritative **`e#` / `m#` / `p#`** appear in **`plasm_context`** replies that **open** the execute session, **append** new picks, or **recover** after a stale/expired binding (see **`_meta.plasm`**). **Steady state:** reuse **`logical_session_ref`** with **`plasm`** / **`plasm_run`**‚Äîdo **not** loop **`plasm_context`** with identical **`seeds`** just to re-fetch DOMAIN; those no-op calls return a short notice **without** repeating the full teaching table (token-saving). If chat history lost the DOMAIN, rely on tool transcripts or open a new logical session / wait for continuity recovery‚Äînot gratuitous same-seed refresh.
     When several distinct **`api`** values load, one primary **`api`** orders the teaching block (alphabetically smallest name); symbols for every loaded capability stay in **one** program surface.
     **`plasm`**: pass **`logical_session_ref`** and one **`program`** string.
