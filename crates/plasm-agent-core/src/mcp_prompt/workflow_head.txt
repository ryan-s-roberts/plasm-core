Plasm is **several MCP tools**; use what you need, in order. **`plasm_context`** opens one logical session; **`plasm`** returns a **dry-run plan** for programs inside it; **`plasm_run`** performs **live execution** after you review that plan; **`discover_capabilities`** finds **`api`** and **`entity`** values that match your intent when you do not already know them. **Skip `discover_capabilities`** when you already know every **`api`/`entity`** pair you need.
     **One user goal → one stable `intent` → one `plasm_context`.** `logical_session_ref` slots (`s0`, `s1`, …) are **per logical goal**, not per API. A second API belongs in the **same** `seeds` array (federation), not a second context open for the same task.
     **`discover_capabilities`**: pass exactly one **`intent`** string (what the user wants to do). The reply is a small fenced **table**: **`api`**, **`entity`**, **`description`**. Each row is one **capability pick** — merge every row you need into **one** **`plasm_context`** **`seeds`** array (several discovery calls may feed the same array). For ambiguous intents, retry with **`typed: true`** for structured disambiguation.
     **`plasm_context`**: pass a stable **`intent`** string and a non-empty **`seeds`** array. Each object is **one capability pick**: **`api`** names the integration, **`entity`** names the resource type (same shape as discovery rows). **List every pick your program needs on the first call**—if the task spans more than one integration, put **all** picks in the **same** **`seeds`** array; that is still **one** session and **one** program surface for **`plasm`** / **`plasm_run`**. Example federated open: `[{"api":"pokeapi","entity":"Pokemon"},{"api":"linear","entity":"Issue"}]`. You may use **`entry_id`** instead of **`api`** on each object. Do not shrink **`seeds`** to “narrow” the session, and do not rotate **`intent`** every user message.
     **Adding picks:** call again when you discover **additional** capability picks you did not know at open time (append new `{api, entity}` rows—still additive).
     **Symbols:** authoritative **`e#` / `m#` / `p#`** appear in **`plasm_context`** replies that **open** the execute session, **append** new picks, or **recover** after a stale/expired binding (see **`_meta.plasm`**). **Steady state:** reuse **`logical_session_ref`** with **`plasm`** / **`plasm_run`**—do **not** loop **`plasm_context`** with identical **`seeds`** just to re-fetch DOMAIN; those no-op calls return a short notice **without** repeating the full teaching table (token-saving). If chat history lost the DOMAIN, rely on tool transcripts or open a new logical session for a **new goal**—not gratuitous same-seed refresh.
     When several distinct **`api`** values load, one primary **`api`** orders the teaching block (alphabetically smallest name); symbols for every loaded capability stay in **one** program surface.
     **`plasm`**: pass **`logical_session_ref`** and one **`program`** string.
